# README Audit Checklist

## Phase 1: Context Setting

- [ ] One-line description present
- [ ] Problem statement clear
- [ ] Solution approach stated
- [ ] Target audience identified
- [ ] Prerequisites listed

## Phase 2: Anti-Patterns

- [ ] No "awesome" or hype language
- [ ] No marketing speak
- [ ] No unsubstantiated claims
- [ ] No emoji overload
- [ ] No redundant badges

## Phase 3: Badges (If Present)

- [ ] Badges are functional (links work)
- [ ] Badges provide value (CI status, coverage, version)
- [ ] No decorative-only badges
- [ ] Badge order is logical
- [ ] Badges are current

## Phase 4: Structure

- [ ] Installation section present
- [ ] Quick start/usage section present
- [ ] Configuration documented
- [ ] API surfaces explained
- [ ] Contributing guidelines linked (if open source)

## Phase 5: Essence

- [ ] Can understand purpose in 30 seconds
- [ ] Can get started in 5 minutes
- [ ] Links to detailed docs work
- [ ] Examples are runnable
- [ ] Contact/support path clear
