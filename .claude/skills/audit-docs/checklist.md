# Documentation Audit Checklist

## Phase 1: File Presence

- [ ] `docs/` directory exists
- [ ] `docs/1.learn/` contains overview and quickstart
- [ ] `docs/2.guides/` contains usage guides
- [ ] `docs/3.reference/` contains API reference
- [ ] All files have proper frontmatter

## Phase 2: Overview Document

- [ ] States what the application does
- [ ] Explains the problem it solves
- [ ] Lists key features
- [ ] Identifies target users
- [ ] Links to quickstart

## Phase 3: Quickstart Document

- [ ] Prerequisites listed
- [ ] Installation steps clear
- [ ] First run instructions complete
- [ ] Expected output shown
- [ ] Common issues addressed

## Phase 4: Concepts Document

- [ ] Core concepts explained
- [ ] Domain model described
- [ ] API surfaces documented
- [ ] Authentication explained
- [ ] Key workflows covered

## Phase 5: Architecture Document

- [ ] System diagram present
- [ ] Layer responsibilities explained
- [ ] Data flow documented
- [ ] Integration points described
- [ ] Deployment topology covered

## Phase 6: Guides

- [ ] Common tasks have guides
- [ ] Guides are task-oriented
- [ ] Examples are complete and runnable
- [ ] Error scenarios addressed
- [ ] Best practices included

## Phase 7: API Reference

- [ ] All endpoints documented
- [ ] Request/response examples present
- [ ] Error codes listed
- [ ] Authentication requirements clear
- [ ] Rate limits documented (if applicable)

## Phase 8: Frontmatter Quality

- [ ] Title is descriptive
- [ ] Description summarizes content
- [ ] Author specified
- [ ] Published date present
- [ ] Tags are relevant
